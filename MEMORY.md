## 设备参数调整原则（2025-03-17）

**不确定范围时，先查后设**
- 调整色温/亮度等参数前，**必须先查询设备状态**
- 确认设备实际支持的范围（最小值、最大值）
- 再根据查询结果设置目标值
- **禁止**拍脑袋使用参考值或假设值

**教训记录：**
- 错误：大哥要求"色温调到最低"，我直接假设2200K是最低值执行
- 结果：设备实际支持到100K/1K，执行错误
- 正确做法：先 `status` 查询，获取实际范围，再设置

**实现方案：**
- FFmpeg 技能新增 `scripts/select-device.sh` 脚本

- 内置设备关键词：`FaceTime`、`Built-in`、`Internal`
- 不含上述关键词的设备视为外置设备，优先选择

## Meeting-Assistant 长录音分割功能（2025-03-25 已完成）

**需求背景：**
- 腾讯云 ASR 限制：单次最多 3 分钟音频
- 会议录音可能长达 1 小时，需要自动分割处理

**实现方案：**
- 新增 `scripts/split-audio.sh` 脚本
  - 使用 ffprobe 检测音频时长
  - 超过 3 分钟自动分割为 2 分 50 秒的小段（留 10 秒缓冲）
  - 使用 ffmpeg 快速分割
- 修改 `scripts/process-meeting.sh`
  - 录音完成后自动检测时长
  - 长音频自动分割 → 逐段转录 → 合并结果
  - 短音频直接转录

**分割策略：**
- 每段 170 秒（2分50秒），避免刚好触达 3 分钟限制
- 使用 `-c:a libopus` 重新编码（确保精确切割点）
- 分段转录后合并为完整转录文本

**文件变更：**
- `skills/meeting-assistant/scripts/split-audio.sh` - 新增分割脚本
- `skills/meeting-assistant/scripts/process-meeting.sh` - 集成分割逻辑
- `skills/meeting-assistant/SKILL.md` - 添加长录音处理文档

**测试验证：**
- 短音频（15秒）→ 不分割，直接转录 ✅
- 长音频（5分钟）→ 分割为2段 ✅
